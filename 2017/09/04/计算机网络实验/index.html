计算机网络实验

Posted at 2017-09-04 计算机网络 

实验一

1.实验目标

  • 设计如图所示的网络，并进行IP地址和路由表的配置

  • 路由器的每一个端口对应一个子网,子网地址为:11.51.15.0~11.51.21.0
  • 终端及其设备端口分配IP地址,并保证连通。

2.实验方法

  1. 使用交叉线,从路由器 R0的FastEthernet1/0端口连接到路由器R1的 FastEthernet1/0端口。设置 R0 的 FastEthernet1/0 端口 IP 为 11.51.15.1 ,子网掩码为 255.255.255.0 。设置 R1 的FastEthernet1/0端口IP为11.51.15.2,子网掩码为255.255.255.0

  2. 使用DTE串口线,从路由器R1的 Serial2/0端口连接到路由器 R2的Serial2/0端口。设置R1的Serial2/0 端口 IP 为 11.51.16.1 ,子网掩码为 255.255.255.0 。设置 R2 的 Serial2/0 端口 IP 为11.51.16.2,子网掩码为255.255.255.0。设置DCE端时钟频率为64000Hz.

  3. 使用交叉线,从路由器 R2的FastEthernet1/0端口连接到路由器R3的 FastEthernet0/0端口。设置 R2 的 FastEthernet1/0 端口 IP 为 11.51.17.1 ,子网掩码为 255.255.255.0 。设置 R3 的FastEthernet0/0端口IP为11.51.17.2,子网掩码为255.255.255.0

  4. 使用DTE串口线,从路由器R3的 Serial2/0端口连接到路由器 R4的Serial2/0端口。设置R3的Serial2/0 端口 IP 为 11.51.18.1 ,子网掩码为 255.255.255.0 。设置 R4 的 Serial2/0 端口 IP 为11.51.18.2,子网掩码为255.255.255.0。设置DCE端时钟频率为64000Hz.

  5. 使用直通线,从路由器R1 的 FastEthernet0/0端口连接到交换机 S1 的 FastEthernet1/1端口。设置 R1 的 FastEthernet0/0 端口 IP 为 11.51.19.1 ,子网掩码为 255.255.255.0 。设置 S1 的FastEthernet1/1端口为开启。

  6. 使用直通线,从交换机 S1的FastEthernet2/1 端口连接到终端设备 PC1的FastEthernet0。设置S1的 FastEthernet2/1端口为开启。设置PC1的网关为 11.51.19.1,FastEthernet0端口 IP 为11.51.19.2,子网掩码为255.255.255.0

  7. 使用直通线,从路由器R2 的 FastEthernet0/0端口连接到交换机 S2 的 FastEthernet1/1端口。设置 R2 的 FastEthernet0/0 端口 IP 为 11.51.20.1 ,子网掩码为 255.255.255.0 。设置 S1 的FastEthernet1/1端口为开启。

  8. 使用直通线,从交换机 S2的FastEthernet2/1 端口连接到终端设备 PC2的FastEthernet0。设置S2的 FastEthernet2/1端口为开启。设置PC2的网关为 11.51.20.1,FastEthernet0端口 IP 为11.51.20.2,子网掩码为255.255.255.0

  9. 使用直通线,从交换机 S2的FastEthernet3/1 端口连接到终端设备 PC3的FastEthernet0。设置S2的 FastEthernet3/1端口为开启。设置PC2的网关为 11.51.20.1,FastEthernet0端口 IP 为11.51.20.3,子网掩码为255.255.255.0

  10. 使用直通线,从路由器R3 的 FastEthernet1/0端口连接到交换机 S3 的 FastEthernet1/1端口。设置 R1 的 FastEthernet1/0 端口 IP 为 11.51.21.1 ,子网掩码为 255.255.255.0 。设置 S3 的FastEthernet1/1端口为开启。

  11. 使用直通线,从交换机 S3的FastEthernet2/1 端口连接到终端设备 PC4的FastEthernet0。设置S3的 FastEthernet2/1端口为开启。设置PC4的网关为 11.51.21.1,FastEthernet0端口 IP 为11.51.21.2,子网掩码为255.255.255.0

  12. 各个路由器配置相应的路由表，除了直接可以连通的子网之外，其他子网均通过相邻的路由器跳转。例如路由器R0的路由表配置：

    | 目标网络 | 子网掩码 | 下一跳 |
    | :——–: | :———–: | :——–: |
    | 11.51.16.0 | 255.255.255.0 | 11.51.15.2 |
    | 11.51.17.0 | 255.255.255.0 | 11.51.15.2 |
    | 11.51.18.0 | 255.255.255.0 | 11.51.15.2 |
    | 11.51.19.0 | 255.255.255.0 | 11.51.15.2 |
    | 11.51.20.0 | 255.255.255.0 | 11.51.15.2 |
    | 11.51.21.0 | 255.255.255.0 | 11.51.15.2 |

3.实验结果

​ 终端与终端之间、路由器与终端之间、路由器与路由器之间均可以互相Ping通。

4.实验感想

​ 通过仿真配置网络实验,我加深了对路由器、交换机和终端设备之间的关系的理解,掌握了三者之间端口的连接、配置方法,学会了路由表的设置,并且了解了配置各个设备的命令行操作方法，还了解到各种线材的作用，为网络设备的实际操作打下基础。

实验二

1.实验目标

​ 3 位同学一组,设计如图的网络,(3 个路由器连接 A-D 的 4 个子网),并进行配置。要求 ABC 子网对应 3 位同学的学号(如 71115101 对应 11.51.1.0/24),通过静态路由的设置,保证路由器间互相 Ping 通。

2.实验方法

  1. 在运行中打开 Telnet 10.3.0.77 10005

  2. 配置 R5的端口地址:

    1. 设置 FastEthernet0/0：ip address 11.51.15.2 255.255.255.0 (与 R3的 FastEthernet0/1 相连);使用子网二的二号地址.

    2. 设置 FastEthernet0/1：ip address 11.51.12.1 255.255.255.0

      (与 R3的 FastEthernet0/1 相连);使用子网三的一号地址.

  3. 配置静态路由表:

    1. ip route 11.51.13.0 255.255.255.0 11.51.15.1(目标地址是子网一,子
      网掩码是 255.255.255.0,下一跳是子网二的一号地址)
    2. ip route 11.13.4.0 255.255.255.0 11.51.12.2(目标地址是子网四,子
      网掩码是 255.255.255.0,下一跳是子网三的二号地址)

  4. 测试与其他路由器的连接(Ping)

3.实验结果

​ 各个路由器之间可以相互Ping通，路由表无误。

4.实验感想

​ 通过这次实验,自己配置路由器,我对子网划分有了更深入的认识.在实验过程中,配置了子网端口和静态路由表,最后使子网之间能够互相连通.我从这次实验中体会到,要学好这门课,还是要多动手实践,并且要对书本上的知识有更多的思考而不是单纯的记忆.
